# Deploying to Streamlit Community Cloud

This guide explains how to deploy your Temporiq AI application to [Streamlit Community Cloud](https://share.streamlit.io/), a free service for sharing Streamlit apps.

## 1. Prerequisites

- Your application code must be in a public or private GitHub repository.
- Your repository must contain a `requirements.txt` file listing all Python dependencies.

## 2. Sign in to Streamlit Community Cloud

1.  Go to share.streamlit.io.
2.  Sign in with your GitHub account and authorize Streamlit to access your repositories.

## 3. Deploy the App

1.  From your workspace, click **"New app"**.
2.  **Repository**: Select the GitHub repository containing your app.
3.  **Branch**: Select the branch you want to deploy (e.g., `main`).
4.  **Main file path**: Set this to the path of your `app.py` file **relative to the root of your GitHub repository**.
    -   **Scenario A:** If `app.py` is visible at the top level of your repository, the path is simply `app.py`.
    -   **Scenario B:** If your project code is inside a subfolder (e.g., you see a folder named `Aegis-Time-AI` in your repository), the path must include that folder: `Aegis-Time-AI/app.py`.
    
    The error "main file path does not exist" almost always means the path needs to include a folder name, as in Scenario B.
5.  Click **"Advanced settings..."**.

## 4. Configure Secrets (Environment Variables)

This is the most important step. Your app needs credentials for Supabase and Google Cloud. You will add these in the **Secrets** section.

```toml
# Paste this into the Streamlit Secrets text box

SUPABASE_URL = "your_supabase_url"
SUPABASE_KEY = "your_supabase_anon_key"
ADMIN_EMAIL = "your_admin_email@example.com"

GCP_PROJECT = "your-gcp-project-id"
GCP_LOCATION = "your-gcp-location"

# SMTP Configuration for Password Resets
SMTP_SERVER = "smtp.resend.com"
SMTP_PORT = "587"
SMTP_USERNAME = "resend"
SMTP_PASSWORD = "re_your_resend_api_key"
SMTP_SENDER_EMAIL = "your_email@yourdomain.com"
SUPABASE_SERVICE_ROLE_KEY = "your_supabase_service_role_key"

# The GCP_CREDENTIALS_JSON line must be generated by the helper script below.
```

### How to get the `GCP_CREDENTIALS_JSON` value:

1.  Open the `.json` service account key file you downloaded from Google Cloud.
2.  Select and copy the **entire content** of the file, including the opening `{` and closing `}`.
3.  Paste this content between the triple quotes (`"""..."""`) for the `GCP_CREDENTIALS_JSON` secret.

### ⚠️ Important: How to Fix the "Invalid TOML" Error

When you paste the JSON content, Streamlit will likely show an **"Invalid Format: please enter valid TOML"** error. This is a common issue caused by newline characters (`\n`) inside the `private_key` field of your JSON file.

You must "escape" these characters before pasting. Here are two methods to do that.

#### Method 1: Use the Helper Script (Recommended)

This repository includes a script called `prepare_secrets.py` that handles the conversion for you automatically.

1.  Open your terminal in the project directory.
2.  Run the script, passing the path to your downloaded GCP key file as an argument:
    ```bash
    python prepare_secrets.py /path/to/your/key.json
    ```
3.  The script will print a block of text to your terminal. Copy the entire output from `--- BEGIN COPY ---` to `--- END COPY ---`.
4.  Paste this corrected text into the Streamlit secrets editor inside the triple quotes for `GCP_CREDENTIALS_JSON`. The error will be gone.

#### Method 2: Manual Find and Replace

1.  **Copy** the entire content of your `.json` key file.
2.  **Paste** it into a plain text editor (like VS Code, Notepad, or TextEdit).
3.  Use the **Find and Replace** feature:
    -   Find: `\n`
    -   Replace with: `\\n` (a double backslash and 'n')
4.  **Copy** the newly modified text and paste it into the Streamlit secrets editor.

**Example of the change needed:**
*   **Before (causes an error):**
`"private_key": "-----BEGIN PRIVATE KEY-----\nMIIC...rest of key...\n-----END PRIVATE KEY-----\n",`
*   **After (correctly escaped):**
`"private_key": "-----BEGIN PRIVATE KEY-----\\nMIIC...rest of key...\\n-----END PRIVATE KEY-----\\n",`

### Example of the final Secrets configuration

When you are done, the secrets text box in Streamlit should look exactly like this, but with your actual values. Notice how the entire JSON content is placed inside the triple quotes.

```toml
# Paste this into the Streamlit Secrets text box

SUPABASE_URL = "https://xyz.supabase.co"
SUPABASE_KEY = "ey..."
ADMIN_EMAIL = "your_admin_email@example.com"

GCP_PROJECT = "my-gcp-project-123"
GCP_LOCATION = "us-central1"

# The entire content of your .json key file goes between the triple quotes.
# TOML (the format used here) handles the multi-line string correctly.
GCP_CREDENTIALS_JSON = """
{
  "type": "service_account",
  "project_id": "my-gcp-project-123",
  "private_key": "-----BEGIN PRIVATE KEY-----\\n...your key data with escaped newlines...\\n-----END PRIVATE KEY-----\\n",
  "client_email": "my-service-account@my-gcp-project-123.iam.gserviceaccount.com"
}
"""
```

The `app.py` file is already configured to read this `GCP_CREDENTIALS_JSON` variable, write it to a temporary file, and make it available to the Google Cloud client library.

## 5. Configure Supabase for Production

This is a critical one-time setup step. For user sign-up confirmations and password resets to work, you must tell Supabase your public app URL.

By default, Supabase sends links that point to `localhost`, which will fail in a deployed environment.

1.  **Find your Streamlit App URL:** Once your app is deployed, it will have a public URL like `https://your-app-name.streamlit.app`. Copy this URL.
2.  **Go to your Supabase Project:** Log in to supabase.com and open your project.
3.  **Navigate to URL Configuration:**
    *   In the left sidebar, click the **Authentication** icon (looks like a person).
    *   In the `Configuration` section, select **URL Configuration**.
4.  **Update the Site URL:**
    *   Change the **Site URL** field from the default (`http://localhost:3000`) to your public Streamlit app URL.
    *   Click **Save**.

Now, all authentication emails sent from Supabase will have the correct links.

## 6. Deploy!

After adding the secrets, click **"Save"** and then **"Deploy!"**.

Streamlit will build the environment from your `requirements.txt` file and launch your application. You can view the logs in real-time by clicking the "hamburger" menu (☰) in the top-right corner of your running app and selecting "View logs".
